## Slide 2: 项目背景与目标

**背景：**
- AI视觉技术的快速发展
- 手势识别在人机交互中的广泛应用
- 音乐教育与娱乐的新形式

**目标：**
- 实现摄像头实时手部关键点检测
- 将手指位置映射到虚拟钢琴
- 支持多指同时演奏

---

## Slide 3: 技术方案

**核心技术栈：**
- MediaPipe Hand Landmarker — 手部21关键点检测
- CameraX — 摄像头实时预览与帧处理
- SoundPool — 低延迟音频播放
- Kotlin + Android SDK

**架构：**
```
摄像头帧 → HandLandmarker → 按键检测 → 琴键映射 → 音频播放
                ↓
            OverlayView（骨骼绘制）
```

---

## Slide 4: 核心算法

**按键检测算法：**
1. 获取21个手部关键点坐标
2. 对每根手指，计算指尖(Tip)与掌根(MCP)的欧氏距离
3. 若距离 < 阈值，判定为"按下"
4. 根据指尖X坐标映射到对应琴键

**优势：**
- 计算简单，实时性好
- 支持多指同时检测
- 阈值可调，适应不同手型

---

## Slide 5: 系统实现

**核心组件：**
| 组件 | 职责 |
|------|------|
| HandLandmarkerHelper | 封装MediaPipe推理，返回手部关键点 |
| FingerPressDetector | 分析关键点，判断手指按压状态 |
| PianoKeyMapper | 将X坐标映射为琴键索引和音符 |
| PianoSoundPlayer | SoundPool管理，播放/停止音符 |
| PianoView | 自定义View，绘制钢琴键盘 |

## Slide 8: 技术挑战与解决方案

| 挑战 | 解决方案 |
|------|---------|
| 实时性要求高 | 使用LIVE_STREAM模式，异步推理 |
| 多指同时触发 | SoundPool支持5路同时播放 |
| 手型差异 | 阈值可调，适应不同用户 |
| 横屏适配 | 强制横屏 + 比例布局 |
