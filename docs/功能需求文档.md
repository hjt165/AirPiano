# Air Piano 功能需求文档

## 1. 项目概述

**项目名称：** Air Piano（AI手势识别空气钢琴）

**项目目标：** 基于 MediaPipe Hand Landmarker，通过摄像头实时识别单手手势，将手指位置映射到虚拟钢琴琴键并触发音频播放，实现"空中弹琴"交互。

**技术栈：** Android (Kotlin) + MediaPipe Tasks Vision + CameraX

---

## 2. 功能需求

### 2.1 核心功能

| 功能模块 | 功能描述 | 优先级 |
|---------|---------|--------|
| 手部检测 | 使用前置摄像头实时检测单手21个关键点 | P0 |
| 手指按压判断 | 通过指尖与掌根距离判断手指是否"按下" | P0 |
| 琴键映射 | 根据手指X坐标映射到对应钢琴琴键 | P0 |
| 音频播放 | 按下琴键时播放对应音符声音 | P0 |
| 琴键可视化 | 实时显示钢琴键盘和按下的琴键 | P0 |

### 2.2 辅助功能

| 功能模块 | 功能描述 | 优先级 |
|---------|---------|--------|
| 推理时间显示 | 实时显示当前帧推理耗时 | P1 |
| 当前音符显示 | 实时显示当前按下的音符名称 | P1 |
| 设置弹窗 | 调整检测阈值、跟踪阈值、按键灵敏度、推理后端 | P1 |
| 骨骼绘制 | 在摄像头画面上叠加手部骨骼连线 | P1 |

### 2.3 约束条件

- 仅支持单手识别（`maxNumHands=1`）
- 仅支持前置摄像头
- 强制横屏模式
- 最低支持 Android API 24 (Android 7.0)

## 3. 技术架构

### 3.1 组件图

```
┌─────────────────────────────────────────────┐
│              CameraFragment                 │
│  ┌─────────────────────────────────────┐    │
│  │         CameraX Preview            │    │
│  │         + OverlayView              │    │
│  └─────────────────────────────────────┘    │
│  ┌──────────┐ ┌──────────┐ ┌────────────┐  │
│  │HandLand- │ │Finger    │ │PianoKey    │  │
│  │marker    │ │Press     │ │Mapper      │  │
│  │Helper    │ │Detector  │ │            │  │
│  └──────────┘ └──────────┘ └────────────┘  │
│  ┌──────────────┐ ┌──────────────────────┐  │
│  │PianoSound    │ │PianoView             │  │
│  │Player        │ │(Custom View)         │  │
│  └──────────────┘ └──────────────────────┘  │
└─────────────────────────────────────────────┘
```

### 3.2 数据流

1. CameraX 捕获图像帧
2. HandLandmarkerHelper 执行手部关键点检测
3. FingerPressDetector 分析指尖位置判断按压状态
4. PianoKeyMapper 将指尖X坐标映射为琴键索引和音符名称
5. PianoSoundPlayer 播放对应音频
6. PianoView 绘制钢琴键盘和按压状态

### 3.3 关键算法

**按键检测算法：**
- 对每个手指，计算指尖(Tip)与掌根(MCP)的欧氏距离
- 若距离 < 阈值（默认0.07），判定为"按下"状态
- 五根手指独立检测，支持同时按多个键

---

## 4. 音频系统

- 使用 SoundPool 支持同时播放多个音符（最大5路）
- 音频文件格式：WAV (44.1kHz, 16-bit, mono)
- 预加载56个音符（C0-B7）
- 当前使用静音占位文件，可替换为真实钢琴音色
