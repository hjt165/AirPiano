# Air Piano UI设计说明

## 1. 设计理念

Air Piano 采用深色主题设计，强调摄像头画面的可见性，钢琴条作为辅助UI位于底部，不抢占视觉焦点。

## 2. 布局设计

### 2.1 整体布局（横屏）

```
┌──────────────────────────────────────────────┐
│                                              │
│           Camera Preview (75%)               │
│                                              │
│                    ⚙ (settings icon)         │
├──────────────────────────────────────────────┤
│ 推理时间: 32ms      │      当前音符: C4       │
├──────────────────────────────────────────────┤
│ [C4] [D4] [E4] [F4] [G4] [A4] [B4] [C5].. │
│  ♪    ♪    ♪    ♪    ♪    ♪    ♪    ♪      │
└──────────────────────────────────────────────┘
```

- **上部 75%**：摄像头预览 + 手部骨骼叠加层 + 设置按钮
- **下部 25%**：信息栏 + 钢琴条

### 2.2 信息栏

- 左侧：推理时间（灰色文字）
- 右侧：当前音符名称（绿色高亮）

### 2.3 钢琴条 (PianoView)

- **白键**：14个，显示C4-B5共两个八度
- **黑键**：按标准钢琴排列（E-F和B-C之间无黑键）
- **按压状态**：按下时显示绿色渐变（#00BFA5 → #00897B）
- **键名标注**：每个白键底部显示音符名称

## 3. 色彩方案

| 元素 | 颜色 | 用途 |
|------|------|------|
| 背景 | #121212 | 主背景色 |
| 钢琴条背景 | #1A1A1A | 底部区域 |
| 信息栏 | #222222 | 状态信息区 |
| 白键 | #F5F5F5 / #E8E8E8 | 交替明暗 |
| 黑键 | #1A1A1A | 黑色琴键 |
| 按压高亮 | #00BFA5 | 按下时渐变色 |
| 主色调 | #007F8B | MediaPipe品牌色 |
| 强调色 | #00BFA5 | 音符显示、按键高亮 |
| 文字主色 | #FFFFFF | 主要文字 |
| 文字辅色 | #AAAAAA | 次要文字 |

## 4. 设置弹窗

### 4.1 弹窗样式
- BottomSheetDialogFragment（底部弹出）
- 深色背景 #1E1E1E
- 圆角顶部

### 4.2 可调参数
- **检测阈值**（Detection）：0.10 - 0.90，步长0.10
- **跟踪阈值**（Tracking）：0.10 - 0.90，步长0.10
- **按键灵敏度**（Press）：0.01 - 0.20，步长0.01
- **推理后端**：CPU / GPU 单选

### 4.3 交互方式
- +/- 按钮调整数值
- 单选按钮选择后端
- "应用"按钮确认并关闭

## 5. 字体规范

| 用途 | 字号 | 颜色 |
|------|------|------|
| 推理时间 | 12sp | #AAAAAA |
| 当前音符 | 12sp | #00BFA5 |
| 琴键名称 | 自适应 | #888888 |
| 设置标题 | 18sp | #FFFFFF |
| 设置标签 | 13sp | #AAAAAA |
| 设置数值 | 15sp | #FFFFFF |

## 6. 适配说明

- 强制横屏，无需适配竖屏
- 使用 `layout_weight` 按比例分配空间
- PreviewView 使用 `fillStart` 缩放模式
- CameraX 使用 4:3 宽高比
