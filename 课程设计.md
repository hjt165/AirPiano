## 一、课程设计交付

### 1.1 功能需求文档(20%)

项目开发需求文档，用于明确项目定位、功能边界、开发标准。

**标准撰写目录：**

1. **项目概述**：开发目标；
2. **功能需求分析**：整体功能介绍、核心功能说明；分模块详细描述每个功能的操作流程、交互效果；
3. **用户需求与角色分析**：用户使用场景；
4. **非功能需求**：性能需求（可配合测试报告说明）。

**示例：**

```
本文档为空气钢琴项目核心功能需求规范，用于明确本项目开发目标、功能边界、用户适配场景及性能开发标准，为产品设计、开发、测试、验收提供统一依据，规避需求偏差、功能冗余、性能不达标等问题，保障项目落地一致性。
1. 项目概述
1.1 开发目标
本项目为AI手势识别空气钢琴系统，核心开发目标为：基于摄像头视觉识别技术，搭建无实体键盘、隔空手势弹奏的智能钢琴交互系统。依托机器视觉捕捉用户手部、指尖动作，映射标准钢琴键位发声，无需任何实体键盘、外接设备，实现随时随地隔空弹钢琴。项目旨在打破传统钢琴设备空间、成本限制，适配零基础音乐爱好者、少儿音乐启蒙、休闲娱乐使用场景，兼具趣味互动与简易教学能力；保障手势识别精准、音色还原真实、交互流畅稳定，支持多模式弹奏、曲目跟练、参数自定义，同时兼容电脑、平板多终端运行，支持后续拓展手势特效、AI评分、联机互动等功能迭代。
2. 功能需求分析
2.1 整体功能介绍
本空气钢琴系统整体功能围绕手势识别、隔空弹奏、音色音效、智能跟练、参数设置、记录查询六大维度搭建，功能架构轻量化、独立性强且联动顺畅。系统通过实时摄像头画面捕捉用户手势动作，智能识别指尖落点与按压手势，精准匹配钢琴键位并触发对应音色；支持多种乐器音色、弹奏速度、音量自定义，内置零基础跟弹教学模式；自动记录用户弹奏记录、练习时长，支持弹奏音频录制回放；界面可视化展示虚拟琴键、手势点位，交互直观易懂。整体满足用户休闲弹奏、音乐启蒙练习、趣味互动娱乐的核心需求，无硬件依赖，低门槛、高适配。
2.2 核心功能说明
系统核心功能聚焦空气钢琴核心使用场景，核心包含六大模块：实时手势识别模块、隔空弹奏发声模块、智能跟弹教学模块、音色音效设置模块、弹奏记录管理模块、系统基础设置模块。六大模块为系统刚需核心，支撑空气钢琴完整交互闭环，所有功能围绕隔空弹奏核心体验展开，严格界定项目功能边界，无冗余无效功能。
2.3 分模块详细功能描述
2.3.1 实时手势识别模块
功能概述：调用设备摄像头，实时捕捉用户手部姿态、指尖位置，精准识别悬空按压、抬手等弹奏手势，过滤无效手势、画面干扰，为隔空弹奏提供核心数据支撑，保障识别稳定性与精准度。
操作流程：
1、用户进入空气钢琴系统，授权开启设备摄像头，系统自动完成画面初始化、环境自适应校准；
2、界面展示可视化88键/简易49键虚拟琴键区域，自动识别用户双手位置，标记指尖点位；
3、用户在对应琴键悬空区域做出按压手势，系统实时识别手势动作，判定有效弹奏指令；
4、自动过滤挥手、晃手、画面杂物等无效干扰动作，无操作时静默监测，降低设备功耗。
交互效果：
1、摄像头启动快速无卡顿，画面实时高清渲染，手势点位实时跟随指尖移动；
2、有效弹奏时对应虚拟琴键高亮变色、动态按压反馈，视觉直观；
3、手势识别失败、摄像头未授权时，页面弹出文字提示及解决方案引导；
4、光线偏弱、画面轻微遮挡场景下，识别依然稳定，无频繁误触、漏触问题；
5、支持自动适配屏幕尺寸，虚拟琴键区域自适应缩放。
......
4. 非功能需求
4.1 性能需求
为保障空气钢琴手势识别精准、弹奏流畅、运行稳定，适配日常休闲、练习使用场景，系统需满足以下性能标准，所有性能指标为上线验收硬性标准：
1. 响应速度需求
摄像头启动加载时间≤1s，手势识别响应延迟≤30ms，手势动作与发声、界面反馈完全同步；页面切换、音色切换、曲谱加载响应时间≤0.8s；录制音频保存、回放加载无延迟，无卡顿、空白、音画不同步异常。
2. 识别精准性能需求
常规室内光线环境下，手势识别准确率≥98%，无频繁误触、漏触；支持双手同时弹奏、和弦弹奏识别，多指尖同步动作识别无错乱；轻微光线变化、背景杂物干扰下，识别稳定性不受影响。
3. 运行性能需求
系统运行内存占用低，前台持续运行无卡顿、闪退、掉帧问题；长时间连续弹奏30分钟以上，无识别失效、音色失真、程序卡死现象；曲谱资源、录制文件存储不占用过多设备空间。
4. 稳定性需求
系统日常运行稳定率≥99.9%，无报错、崩溃、自动退出问题；摄像头授权异常、光线过暗等特殊场景下，系统友好提示，不强制闪退；异常中断后可快速重启恢复正常使用。
```

### 1.3 作品(50%)

可正常运行的完整Android项目成品，为课程设计核心考核内容。

**提交内容明细：**

- 项目安装包：APK安装包，可直接在Android手机安装运行。

**功能验收标准：**

- 核心功能全部实现，运行流畅，无闪退、无崩溃、无核心逻辑bug；
- 页面布局整洁，交互逻辑合理，按钮、跳转、弹窗等交互正常；
- 适配主流手机屏幕，无严重适配错乱问题。

### 1.4 演讲PPT(10%)

用于课程设计现场汇报，简洁直观、重点突出。

**PPT标准内容结构：**

2. 项目难点与解决思路：重点展示小组问题解决能力；
3. 组员分工与贡献说明：明确每个人的工作内容。
