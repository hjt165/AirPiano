## 幻灯片 2：项目概述

**一句话描述：** 用手指在空中弹钢琴，无需任何物理乐器。

**核心功能：**
- 前置摄像头实时手部追踪
- 手指敲击动作检测
- 对应钢琴音符播放
- 琴键可视化叠加层

**技术栈：** Android / Kotlin / MediaPipe / CameraX

## 幻灯片 3：技术架构

**数据流图：**

```
摄像头 (CameraX)
    ↓ ImageProxy
MediaPipe Hand Landmarker
    ↓ 21 个手部关键点
敲击检测状态机
    ↓ TAP_DOWN / HOLD / RELEASE
    ├── 音符播放 (MediaPlayer)
    └── 琴键 UI 更新 (OverlayView)
```

**关键组件：**
- GestureRecognizerHelper：MediaPipe 推理封装
- AirPianoManager：状态机 + 音频播放
- PianoKeyDrawer：琴键绘制
- OverlayView：手部关键点 + 琴键叠加

**MediaPipe Hand Landmarker：**
- 21 个手部关键点（手腕、指关节、指尖）

**关键点索引：**
- 0: 手腕

- 12: 中指尖

## 幻灯片 5：核心功能 2 — 敲击检测

**状态机设计：**

```
UNKNOWN ──→ TAP_DOWN ──→ HOLD ──→ RELEASE
   ↑                                    │
   └────────────────────────────────────┘
```

**检测原理：**
- 计算指尖 Y 轴偏移量相对于手腕的变化
- offsetDiff = tipOffset - palmOffset
- 阈值判断状态转换

**为什么用相对偏移？**
- 消除手部整体移动的干扰
- 只检测手指的独立敲击动作

**手指-音符映射：**

|------|------|------|--------|------|
| Do(C) | Re(D) | Mi(E) | Fa(F) | Sol(G) |

## 幻灯片 8：挑战与解决方案

### 挑战 1：手指位置抖动
**问题：** 摄像头噪声导致关键点坐标帧间跳动
**方案：** 5 帧加权移动平均平滑（SMOOTHING_FACTOR=0.8）

### 挑战 2：区分有意敲击与自然移动
**问题：** 手部整体移动也会改变指尖坐标
**方案：** 使用相对偏移（指尖 vs 手腕），而非绝对位置

### 挑战 3：声音延迟
**问题：** MediaPlayer 首次播放有加载延迟
**方案：** 预加载所有音符 + seekTo(0) 实现即时重播
